在集成 `rg`（物理检索）、`tree-sitter` / `ast-grep`（语法解析）与 `clangd/LSP`（编译语义）的异构分析系统架构中，**异构结果的归一化（Normalization）是破局上下文爆炸与信噪比失衡的唯一核心解法**。

将其从多源日志拼接，升级为“编译级语义图谱组装”，完整架构分析与落地结论如下：

---

## 一、 核心架构结论：三级漏斗与归一化模型

整套架构的本质，是通过多引擎在不同维度抓取线索，再强行收割至一个具备“空间与语义双重身份”的统一中间表示（IR）中，最终通过“降噪去重”和“预算裁剪”实现精简输出。

```
[rg: 文本正则] ──┐
[tree-sitter] ──┼─► [异构数据归一化 (CodeAtom IR)] ─► [三维去重引擎] ─► [贪心降级裁剪] ─► [最终 LLM Prompt]
[clangd/LSP ] ──┘

```

### 1. 异构源归一化映射规则

不同工具对代码的认知颗粒度不同，必须制定严格的向内转换规则：

| 提取工具 | 认知颗粒度 | 归一化策略（赋予 ID 与物理坐标） |
| --- | --- | --- |
| **`rg` (ripgrep)** | 行级纯文本 | 缺少语义 ID；按匹配行上下扩展 2 行，仅注册物理坐标 `[StartByte, EndByte]`，打上最高降级风险标记。 |
| **`tree-sitter` / `ast-grep**` | 局部语法树节点 | 提取完整函数/类/分支的起始字节；用“`相对路径 + AST 节点类型 + 局部签名`”生成临时 ID。 |
| **`clangd` (LSP)** | 全局语义符号 | 提取精准定义的物理坐标；强制提取并绑定 Clangd 的 **`USR`（统一符号解析 ID）** 作为全局绝对唯一身份ID。 |

---

## 二、 归一化核心结构（CodeAtom IR）

归一化的关键，在于结构体必须同时支撑**空间计算（物理去重）**、**图论计算（语义去重）**与**排版降级（Token 控制）**。

```go
type CodeAtom struct {
    // 1. 语义身份（解决符号交叉依赖与跨文件重复）
    SemanticID  string    // 核心：Clangd USR > FQN (全限定名) > AST Hash
    Name        string    // 符号名称（如 "process_order"）
    Kind        string    // "FUNCTION" | "STRUCT" | "MACRO" | "SNIPPET"

    // 2. 空间坐标（解决同一文件内的区域重叠与包含）
    FilePath    string
    StartByte   int       // 字节级偏移量是区间计算的唯一真理，不用行号
    EndByte     int

    // 3. 多级载荷（解决 Token 窗口爆炸）
    FullContent string    // L0: 完整代码原貌（含注释与内部逻辑）
    Signature   string    // L1: 仅接口签名与 DocString（移除函数体/成员实现）
    Reference   string    // L2: 仅名称与文件行号（如 "struct Order defined in order.h:45"）

    // 4. 治理元数据
    SourceTool  string    // 标注来源："rg" | "ast-grep" | "clangd"
    Priority    float64   // 业务重要度评分：核心 Sink 点 > 直接调用链 > 通用基础结构
}

```

## 三、 三维去重引擎（The 3D Deduplication Engine）

数据完全归一化后，在这个结构之上顺序执行三道过滤，彻底清除冗余：

### 1. 物理空间吞并（Sweep-Line 扫描线算法）

针对单文件内 `rg` 碎片与 `tree-sitter` 完整语法块冲突的场景。

* **计算逻辑：** 在每个 `FilePath` 下，将所有 Atom 按 `StartByte` 排序。时间复杂度为 $O(N \log N)$ 的区间扫描。
* **吞并规则：**
* **完全包含：** 若 `ast-grep` 提取的函数 $A [100, 500]$ 包含了 `rg` 抓到的行片段 $B [200, 250]$，**无条件丢弃 $B$，保留 $A$**。
* **交错重叠：** 若两个 AST 截取块发生交错（如跨行的复杂的逻辑块），利用 `tree-sitter` 瞬间向上一层定位它们的最近公共祖先节点（LCA），将其合并为一个更大的闭合块。



### 2. 全局语义去重（Symbol ID Collision Resolution）

针对跨文件依赖时，多个函数依赖同一底层 `struct` 或宏定义，导致多次通过 LSP 捞取同一外部符号的场景。

* **计算逻辑：** 建立基于 `SemanticID` (USR) 的全局哈希表。
* **冲突决策：** 当不同切片路径引入了相同 USR 的 Atom 时，只保留一个实例，同时将其 `Priority` 设定为**多次被引用路径中的最高优先级**（热点符号权重提升）。

### 3. 结构层级折叠（Parent-Child Subsumption）

当目标文件同时将 `Class T` (全长 800 行) 和其内部的 `Method T.eval()` (全长 30 行) 作为独立 Atom 抓取时：

* **折叠规则：** 剥离 `Class T` 的 `FullContent`，将父级类**降级为 `Signature` 骨架**（仅展示类变量和成员函数声明），同时保留子级 `Method T.eval()` 的 `FullContent`。在保留作用域上下文的同时，省去 90% 不相关的成员实现代码。

## 四、 上下文预算治理：基于优先级的贪心降级

去重完毕后，不再使用基于字符数的物理截断，而是将 Prompt 构建转化为标准的**0-1 背包优化问题的变种（动态降级背包）**。

设定 LLM 窗口可用最大 Token 预算为 $T_{\text{max}}$，目标是最大化上下文语义价值：


$$\max \sum_{i=1}^{n} \text{Priority}_i \times \text{StateLevel}_i \quad \text{s.t.} \sum_{i=1}^{n} \text{Tokens}(i, \text{StateLevel}_i) \le T_{\text{max}}$$

| 评估阶段 | 动作与降级规则 | Token 消耗比 | 适用策略 |
| --- | --- | --- | --- |
| **Phase 1: 贪心装填** | 对所有 Atom 按 `Priority` 从高到低降序遍历，优先填入 `FullContent`。 | $100\%$ | 核心函数、直接漏洞污染源（Source/Sink）、直接主调函数。 |
| **Phase 2: 优雅降级** | 当剩余 Token 不足以装入下个 Atom 的 `FullContent` 时，**不丢弃该 Atom**，而是降级装入其 `Signature`（剥离实现体）。 | $\approx 15\% - 20\%$ | 二层调用栈以上的辅助函数、外部复杂的结构体定义。 |
| **Phase 3: 极致压缩** | 预算极度枯竭时，将低优先级基础依赖降级为 `Reference`（仅留符号引用坐标）。 | $\approx 2\% - 5\%$ | 标准库包装、全局通用配置文件定义、底层接口契约。 |
| **Phase 4: 末位淘汰** | 连 `Reference` 也无法容纳时，丢弃最末端的 Atom。 | $0\%$ | 权重极低的关联外围代码。 |

## 五、 最终工程结论与效益评估

通过将 `Go` 的并发调度能力与这套“**异构归一化 + 三维去重 + 贪心背包降级**”体系结合，你的系统将达成以下行业顶级的工程指标：

1. **极高的容错与完备性：** 借由 `tree-sitter` 兜底，即使在头文件部分丢失、代码编译不完全通过的环境下，发给大模型的上下文仍然保证 100% 语法结构闭合。
2. **绝对的无噪纯净度：** 彻底杜绝了文本检索的碎片日志与 AST 结构的互相重叠，消除了跨文件依赖的全局重复符号。
3. **精确的成本与延迟控制：** 无论用户输入多庞大的全栈工程代码，LLM 的 Prompt 永远被精确钳制在设定的 Token 阈值（如 16K 或 32K）内，切片归一与去重计算完全在内存中通过 Go 结构体引用运行，处理百余个代码块的延迟通常低于 **50 毫秒**。

这套架构解决了“静态程序分析（SAST）追求穷尽依赖”与“大模型（LLM）追求高信噪比与有限窗口”之间的根本矛盾，是目前大模型工程结合代码安全审计领域的终极落地演进方向。
